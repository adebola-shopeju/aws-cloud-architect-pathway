# Week 4, Day 3 — S3 Static Website Hosting

## Live demo
http://adebola-w4d3-static-site.s3-website.eu-west-2.amazonaws.com

## What I built
Hosted a static website directly from an S3 bucket — no EC2 instance or server involved. Configured a public-read bucket policy, deliberately unblocked public access, and enabled static website hosting with `index.html` as the index document and `error.html` as the error document.

## What I debugged
- Fixed a corrupted HTML file caused by TextEdit saving in Rich Text mode instead of plain text (switched to `nano`).
- Fixed a character encoding issue (emoji and dash rendering as garbled symbols) by adding `<meta charset="UTF-8">` to the HTML head.
